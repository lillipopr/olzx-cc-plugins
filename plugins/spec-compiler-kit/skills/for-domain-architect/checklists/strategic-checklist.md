# 战略设计检查清单

> **对应评分标准**：[strategic-scoring.md](../scoring/strategic-scoring.md)
> **满分**：100 分
> **及格分**：60 分

---

## 使用说明

在完成战略设计后，逐项检查本清单。根据每个检查项的评分标准计算得分。

**评分方式**：
- 完全符合：满分
- 基本符合：60-80% 分数
- 不符合：0 分

---

## 一、业务能力分析（35 分）

### 1.1 业务能力树（15 分）

- [ ] **完整性**（5 分）
  - [ ] 覆盖 PRD 中的所有核心业务能力
  - [ ] 无明显遗漏

- [ ] **层级结构**（5 分）
  - [ ] 能力层级清晰
  - [ ] 不超过 3 层

- [ ] **命名规范**（5 分）
  - [ ] 能力名称使用动词+名词格式
  - [ ] 语义明确

### 1.2 能力分级（10 分）

- [ ] **分级正确性**（5 分）
  - [ ] 核心域/支撑域/通用域分类正确
  - [ ] 核心域识别准确

- [ ] **说明充分**（3 分）
  - [ ] 每个能力都有清晰的说明
  - [ ] 说明内容充分

- [ ] **战略重要性**（2 分）
  - [ ] 战略重要性评估合理
  - [ ] 高重要性对应核心域

### 1.3 能力与团队对齐（10 分）

- [ ] **责任明确**（4 分）
  - [ ] 每个能力都有明确的负责团队
  - [ ] 责任划分清晰

- [ ] **上下游关系**（4 分）
  - [ ] 上下游关系定义清晰
  - [ ] 依赖关系明确

- [ ] **完整性**（2 分）
  - [ ] 所有核心能力都有对齐信息
  - [ ] 无遗漏

---

## 二、限界上下文划分（40 分）

### 2.1 上下文识别（15 分）

- [ ] **边界清晰**（6 分）
  - [ ] 上下文边界清晰
  - [ ] 职责明确

- [ ] **与能力对应**（5 分）
  - [ ] 上下文与业务能力正确对应
  - [ ] 一致性良好

- [ ] **类型正确**（4 分）
  - [ ] 核心域/支撑域/通用域分类正确
  - [ ] 类型判断有依据

### 2.2 上下文边界图（10 分）

- [ ] **可视化清晰**（4 分）
  - [ ] 图形清晰易懂
  - [ ] 格式规范

- [ ] **核心实体完整**（4 分）
  - [ ] 每个上下文都列出核心实体
  - [ ] 实体列表完整

- [ ] **边界明确**（2 分）
  - [ ] 上下文边界用图形明确标示
  - [ ] 边界清晰可辨

### 2.3 上下文数量合理（5 分）

- [ ] **数量合理**（5 分）
  - [ ] 上下文数量适中（3-7 个为佳）
  - [ ] 数量与业务规模匹配

### 2.4 上下文命名（5 分）

- [ ] **语义明确**（3 分）
  - [ ] 上下文名称能表达其职责
  - [ ] 命名具有业务含义

- [ ] **术语统一**（2 分）
  - [ ] 使用统一的业务术语
  - [ ] 术语在整个文档中一致

### 2.5 限界上下文原则验证（5 分）

- [ ] **单一职责**（2 分）
  - [ ] 每个上下文只负责一个业务领域
  - [ ] 职责单一

- [ ] **最小知识**（2 分）
  - [ ] 上下文间依赖最小化
  - [ ] 依赖关系合理

- [ ] **独立演化**（1 分）
  - [ ] 上下文可独立演化
  - [ ] 无强耦合

---

## 三、上下文映射（25 分）

### 3.1 上下文关系图（8 分）

- [ ] **关系完整**（4 分）
  - [ ] 所有上下文间的关系都有标示
  - [ ] 无遗漏关系

- [ ] **关系类型正确**（4 分）
  - [ ] 上游/下游/合作关系分类正确
  - [ ] 关系类型准确

### 3.2 集成模式定义（10 分）

- [ ] **模式选择正确**（5 分）
  - [ ] O/C/D/ACL/PL/CF 模式选择正确
  - [ ] 模式选择有依据

- [ ] **说明充分**（3 分）
  - [ ] 每个集成模式都有清晰说明
  - [ ] 说明内容充分

- [ ] **理论依据**（2 分）
  - [ ] 引用 DDD 集成模式理论
  - [ ] 有理论支撑

### 3.3 防腐层设计（7 分）

- [ ] **识别正确**（3 分）
  - [ ] 需要防腐层的地方都识别出来了
  - [ ] 识别准确

- [ ] **转换逻辑清晰**（2 分）
  - [ ] 转换逻辑描述清晰
  - [ ] 逻辑可理解

- [ ] **实现位置明确**（2 分）
  - [ ] 实现位置已明确
  - [ ] 位置可确定

### 评分结果

- [ ] **≥ 90 分**：优秀，可以交付
- [ ] **80-89 分**：良好，建议微调
- [ ] **70-79 分**：合格，建议改进
- [ ] **60-69 分**：及格，必须改进
- [ ] **< 60 分**：不合格，必须重新设计

### 改进建议
