# /spec - 规格编译器

## 功能

规格编译器主入口，根据用户意图智能路由到对应工作流。

## 使用方式

```
/spec              # 交互式选择场景
/spec new          # 首次功能建设
/spec iter         # 功能迭代
/spec fix          # Bug 修复
/spec offline      # 功能下线
/spec review       # 审查规格文档
```

## 执行流程

1. **识别场景**
   - 如果用户指定了场景，直接路由
   - 否则询问用户选择

2. **识别架构**
   - 检测项目类型
   - 确定涉及的端

3. **路由到对应的 Agent 和 Skill**
   - 根据场景调用对应的 senior-xxx/AGENT.md
   - 引用对应的 skills/for-{role}/SKILL.md
   - 执行对应 workflow

## 场景选择提示

```
请选择场景：
1. 首次功能建设 - 完整流程（PRD → DDD → 建模 → 工件）
2. 功能迭代 - 基于已有 PRD 增量更新
3. Bug 修复 - 补充用例 → 推导修复
4. 功能下线 - 逆向清理
5. 审查文档 - 检查规格文档完整性和一致性

请输入数字或场景名称：
```
